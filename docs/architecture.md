# Архитектура приложения

Crypto Telegram Bot построен как Python-приложение с двумя запускаемыми
процессами и общей переиспользуемой логикой внутри `src/`.

Основная архитектурная идея — отделить пользовательский интерфейс Telegram,
сбор рыночных данных, доступ к базе данных и расчёт аналитики друг от друга.
Это позволяет менять обработчики бота, алгоритм анализа или структуру хранения
данных без полного переписывания проекта.

## Запускаемые процессы

| Процесс | Точка запуска | Назначение |
| --- | --- | --- |
| Telegram-бот | `src.bot.bot_main` | принимает команды и callback-запросы пользователей |
| Сборщик данных | `src.data_collector.collector` | получает котировки, запускает анализ и отправляет уведомления |

Оба процесса используют общую конфигурацию из `src.config` и общий слой доступа
к SQLite через `src.database.db_manager`.

## Общая схема взаимодействия

Telegram-бот отвечает за команды, меню и подписки. Сборщик отвечает за внешние
котировки, аналитический расчёт и рассылку алертов. SQLite выступает общей
точкой хранения состояния.

Подробный порядок вызовов между пользователем, ботом, сборщиком, базой данных,
Kraken API и Telegram API описывается Sequence диаграммами.

## Границы модулей

| Модуль | Ответственность |
| --- | --- |
| `src.config` | загрузка `.env`, получение `BOT_TOKEN`, `ADMIN_ID` и `DB_PATH` |
| `src.database.schema` | создание таблиц SQLite |
| `src.database.seed` | первичное заполнение справочника активов |
| `src.database.db_manager` | функции чтения и записи данных без раскрытия SQL остальным модулям |
| `src.analytics.analyzer` | расчёт тренда и признака аномальной волатильности |
| `src.data_collector.collector` | получение данных с Kraken, сохранение цен, запуск анализа и уведомлений |
| `src.bot.bot_main` | создание `Bot`, `Dispatcher`, регистрация команд и запуск polling |
| `src.bot.handlers` | обработчики команд, callback-запросов и текстовых кнопок |
| `src.bot.keyboards` | построение reply- и inline-клавиатур |
| `src.bot.messages` | тексты сообщений и приветствия |

## Слой данных

SQLite-база хранит шесть основных групп данных:

| Таблица | Назначение |
| --- | --- |
| `assets` | справочник отслеживаемых торговых пар |
| `price_history` | история полученных цен и объёмов |
| `users` | зарегистрированные пользователи Telegram |
| `subscriptions` | связи пользователей с активами |
| `analytics` | результаты анализа тренда и аномалий |
| `notifications_archive` | архив отправленных уведомлений |

Модуль `db_manager` является единой точкой доступа к этим таблицам для бота и
сборщика. Обработчики Telegram не выполняют SQL-запросы напрямую.

## Основные взаимодействия

В архитектуре выделяются три группы взаимодействий:

* пользовательские сценарии Telegram-бота;
* сбор рыночных данных и сохранение истории цен;
* анализ аномалий и отправка уведомлений подписчикам.

Эти взаимодействия не дублируются пошагово в тексте, потому что после
добавления диаграмм их лучше читать по Use-case и Sequence нотациям.

## Переиспользуемый компонент

Пакет `analytics` оформлен как отдельный Python-пакет через `pyproject.toml`.
Его публичная задача — принимать историю цен и возвращать результат анализа:

```text
(predicted_direction, is_anomaly, anomaly_description)
```

Этот компонент не зависит от Telegram-интерфейса и не должен знать, как именно
результат будет показан пользователю.

## Текущие архитектурные ограничения

* SQLite используется как локальное хранилище, поэтому проект ориентирован на
  локальный запуск и учебную воспроизводимость.
* Сборщик импортирует объект `bot` из `src.bot.bot_main`, чтобы отправлять
  уведомления. Это рабочий вариант для небольшого проекта, но в более крупной
  системе отправку сообщений можно было бы вынести в отдельный сервис.
* Работа с SQLite выполняется синхронно. Для текущего масштаба этого достаточно,
  но при росте нагрузки потребуется пересмотреть модель доступа к данным.
* Алгоритм аналитики намеренно простой: процентное изменение и пересечение
  быстрых/медленных скользящих средних. Он нужен для демонстрации процесса, а не
  для профессионального трейдинга.
